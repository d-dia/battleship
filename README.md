# Игра «Морской бой»

Консольная игра против компьютера. Игрок стреляет по полю компьютера, компьютер отвечает и при попадании включает умное добивание, простреливая соседние клетки, пока не потопит корабль. Флот на обоих полях расставляется случайно. Партию можно сохранить в JSON и при следующем запуске продолжить с того же места.

## Что умеет игра

- Игра на поле 10 на 10 со случайной расстановкой флота
- Умное добивание у компьютера, после попадания он бьет по соседним клеткам
- Отмена последнего выстрела игрока командой `undo` с откатом счетчиков
- Сохранение партии в JSON и загрузка ее при запуске программы
- Статистика по флоту компьютера, число оставшихся кораблей и их размеры
- Просмотр своего поля и списка команд в любой момент

## Структуры данных и алгоритмы

- Двумерные списки для поля игрока и поля компьютера
- Стек для отмены последнего выстрела игрока
- Очередь клеток для алгоритма добивания корабля компьютером
- Бинарное дерево поиска свободных клеток. Ключ это номер строки, значение это множество еще не простреленных столбцов. Дерево строится сбалансированным, поэтому поиск по строке быстрый
- Бинарный поиск, проверяет, остались ли на плаву корабли заданного размера
- Сортировка слиянием для вывода размеров кораблей в статистике

## Структура проекта

- `structures.py` хранит классы `Stack`, `Queue`, `Node`, `BST` и функции `binary_search`, `merge_sort`
- `field.py` хранит работу с полем, расстановку кораблей, вывод поля, логику добивания, создание и восстановление партии
- `saving.py` хранит функции `save_to_file` и `load_from_file` для работы с JSON
- `main.py` хранит игровой цикл и занимается обработкой команд

## Использование программы

При старте программа спрашивает, загрузить ли сохраненную партию. Если ответить н, начинается новая игра. Поле компьютера показывается со скрытыми кораблями, свое поле видно целиком.

Во время партии работают такие команды:

- `<строка> <столбец>` выстрел, например 3 5
- `undo` отменяет последний выстрел игрока
- `stats` показывает статистику по кораблям компьютера
- `save` сохраняет партию в файл
- `myfield` показывает свое поле
- `help` показывает список команд
- `quit` выход

На поле символ ~ означает воду, S свой корабль, X попадание, звездочка промах

## Как устроен код

Класс `Stack` это стек на основе списка с методами `push`, `pop`, `is_empty`. В нем хранится история выстрелов игрока, благодаря чему последний ход можно откатить.

Класс `Queue` это очередь с методами `enqueue`, `dequeue`, `is_empty`. В нее попадают клетки, которые компьютер собирается проверить при добивании корабля.

Классы `Node` и `BST` это узел и бинарное дерево поиска. Узел хранит номер строки и множество свободных столбцов. Дерево умеет вставлять узел `insert`, искать по строке `search`, собирать все свободные клетки `all_cells` и строиться методом `build_balanced`, который берет середину списка строк корнем.

Модуль `field.py` отвечает за игровое поле. Функция `place_ships` случайно расставляет флот, `cells_free` следит, чтобы корабли не касались, `print_field` рисует поле, `find_ship` и `is_sunk` находят корабль и проверяют его потопление, `next_targets` подсказывает клетки для добивания, `build_free_tree` строит дерево свободных клеток, `targets_from_hits` собирает очередь добивания. Функция `new_game` создает игру в виде словаря, а `fix_loaded` после загрузки превращает координаты из JSON списков обратно в кортежи.

Модуль `saving.py` пишет и читает партию в JSON. Дерево свободных клеток и очередь добивания в файл не сохраняются, при загрузке они строятся заново из поля игрока, поэтому файл остается компактным и без лишних данных.

Модуль `main.py` управляет игрой. Функция `start_game` выбирает новую или сохраненную партию, `player_shot` обрабатывает выстрел игрока, `computer_turn` делает ход компьютера, беря клетку сначала из очереди добивания, а если она пуста, то случайную свободную, `undo_move` откатывает последний выстрел, `show_stats` выводит статистику с сортировкой и бинарным поиском. Все объединяет главный цикл в функции `main`.

### Федеёва Диана

- Классы `Node` и `BST`, методы `insert`, `search`, `all_cells`, `build_balanced` (`structures.py`)
- Функции `binary_search` и `merge_sort` (`structures.py`)

### д

- Расстановка флота `place_ships` и проверка соседства `cells_free` (`field.py`)
- Вывод поля `print_field`, поиск корабля `find_ship`, проверка потопления `is_sunk` (`field.py`)

- Создание и восстановление партии `new_game`, `fix_loaded` (`field.py`)

### т

- Сохранение и загрузка партии `save_to_file`, `load_from_file` (`saving.py`)
- Ход компьютера `computer_turn` и выстрел игрока `player_shot` (`main.py`)
